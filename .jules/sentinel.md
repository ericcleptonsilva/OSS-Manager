## 2026-04-26 - [CRITICAL] Fix hardcoded backdoor credentials and auth bypass
**Vulnerability:** Hardcoded email credentials ('ericlobobr.01@gmail.com', 'admin@oss.com') were present in `App.tsx` granting automatic admin privileges on the client side. Additionally, fallback credentials (`admin@oss.com`/`admin`) were used in `services/parseService.ts` if database properties weren't found, allowing authentication bypass if the database was missing admin setup.
**Learning:** Hardcoded emails used as simple "admin overrides" during development were left in production code. Fallback values using the `||` operator for sensitive properties (passwords, emails) create unintentional "default passwords" that can bypass normal authentication mechanisms if a field is unset in the database.
**Prevention:** Never hardcode user emails as a form of authorization or role assignment. Always use structured roles (e.g., explicitly checking `user.role === 'admin'`). For fallback mechanisms during authentication, verify that stored credentials strictly exist (are not empty strings) before allowing a match to proceed.
