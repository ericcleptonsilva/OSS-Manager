## 2025-05-10 - Hardcoded Admin Credentials and Empty String Auth Bypass
**Vulnerability:** The application had hardcoded backdoors allowing specific email addresses (`ericlobobr.01@gmail.com`, `admin@oss.com`) full admin access regardless of the database role. Furthermore, the legacy authentication logic in `services/parseService.ts` (`performCustomLogin`) defaulted stored database credentials to these hardcoded strings, or worse, if removed completely without proper strict checks, could allow authentication bypass if empty database fields were compared to empty string inputs.
**Learning:** Hardcoded overrides in frontend role assignment functions (`App.tsx`) bypass database-backed role security entirely. Additionally, fallback authentication methods relying on raw string comparisons against database fields must guarantee those fields are not empty strings; otherwise, an empty input can match an empty database field, resulting in an unintended successful login (authentication bypass).
**Prevention:** Never use hardcoded emails or passwords as default credentials or role overrides in application code. When performing manual credential comparisons, always enforce strict falsy checks (e.g., `if (storedAdminEmail && ...)`) to ensure empty strings from the database cannot be matched by empty inputs.
