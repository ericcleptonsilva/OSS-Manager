
## 2024-05-24 - Hardcoded Admin Backdoors and Empty Password Bypass
**Vulnerability:** The codebase contained multiple hardcoded admin backdoors allowing `ericlobobr.01@gmail.com` and `admin@oss.com` to bypass authorization and gain admin roles without validating their passwords or their existence as active Parse users. Furthermore, fallback logic for legacy accounts had no explicit checks for empty passwords, meaning if a backend object had an undefined or empty password, a malicious user could potentially authenticate by supplying an empty password during login.
**Learning:** Hardcoding credentials or specific identifying emails as "superusers" in the code creates an unpatchable backdoor if leaked, bypassing the established RBAC system. Fallback custom authentication logic must explicitly handle missing backend data properties to prevent unintentional null/empty string comparisons during validation.
**Prevention:** Never use hardcoded identity checks (`email === '...'`) to grant roles. Rely strictly on roles fetched from the identity provider (Parse Server). When performing custom manual password comparisons as fallbacks, explicitly verify that both the stored secret and the user-provided secret are truthy/non-empty before performing the equality check.
