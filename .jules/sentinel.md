## 2024-05-23 - Credential Exposure in Parse Query Payloads
**Vulnerability:** The application was inadvertently querying for `Team` and `Academy` records without explicitly excluding the `adminPassword` field. While the application mapper functions were successfully stripping out the sensitive fields before rendering to the client state, the underlying database payloads received in the browser via network requests (e.g., from Back4App via Parse SDK) still contained the raw credentials. This allowed anyone inspecting the network traffic to see the passwords.
**Learning:** Returning large, un-filtered queries to the client-side Parse SDK exposes all properties of the document, bypassing application-level sanitation maps. The mapping alone only hides data from the React state, not from the browser's developer tools network tab.
**Prevention:** Always use the `.exclude()` method on Parse queries for known sensitive fields like `adminPassword` before calling `.first()` or `.find()`. This forces the server to drop the fields before transmitting the response payload over the network.
