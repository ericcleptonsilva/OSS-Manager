## 2024-05-24 - [Remove Hardcoded Credentials & Fix Fallback Auth Bypass]
**Vulnerability:** The application contained hardcoded backend fallback admin credentials ('admin@oss.com' and 'admin') and frontend role assignments based on specific email strings ('admin@oss.com' and 'ericlobobr.01@gmail.com'). Additionally, the fallback authentication could be bypassed if the stored credentials were not set (i.e. empty string values), since comparing empty strings with an empty input would incorrectly authorize the user.
**Learning:** Hardcoded credentials should not be present in code for authentication logic. Fallback mechanisms should always explicitly verify that the fetched credentials are not empty strings to prevent authorization bypass.
**Prevention:** Rely entirely on properly configured external databases (e.g., Back4App) and properly provisioned User objects with explicit role claims, rather than using string matching against specific known emails. Ensure all credential comparisons strictly require non-empty truthy values.
