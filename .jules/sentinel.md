## 2025-02-26 - Insecure Direct Object Reference via Parse Queries
**Vulnerability:** The client-side application was fetching full Parse objects (`Team`, `Academy`) without explicitly stripping the `adminPassword` and `adminEmail` fields. While the client-side mappers were omitting these values before rendering the UI, the raw payload still contained the credentials, exposing them to the browser's network tab and potentially malicious users.
**Learning:** Over-fetching data and relying exclusively on client-side mapping to hide credentials is a critical security vulnerability.
**Prevention:** Sensitive fields that are proven to exist on Parse objects (e.g., `adminPassword` on `Team` and `Academy` classes) MUST be explicitly excluded from Parse Server queries using `.exclude()` to prevent credential exposure in client payloads (e.g., in `fetchFullData` and `fetchPublicData`), except where explicitly required for custom server-side authentication like `performCustomLogin`.
