## 2024-04-24 - Authorization Bypass & Hardcoded Backdoor Credentials
**Vulnerability:** The application contained hardcoded authorization bypass checks in `App.tsx` (`ericlobobr.01@gmail.com` and `admin@oss.com`) that granted automatic global admin access. It also contained fallback logic in `services/parseService.ts` that verified empty strings or `admin` against legacy user logins if the standard user check failed.
**Learning:** Overriding RBAC with hardcoded emails is a critical authorization vulnerability. In legacy codebases or rapid prototypes, developers often add these backdoors for ease of testing or migrations. Additionally, when extracting nested credentials without ensuring they exist first (e.g., matching on an empty string for `adminEmail` and `adminPassword` from the database), malicious users could potentially bypass auth entirely by passing empty strings.
**Prevention:** Strictly enforce Role-Based Access Control (RBAC) defined by the backend. Do not leave hardcoded emails or backdoor credentials in production code. Always validate that credential fields retrieved from the database are non-empty before proceeding with legacy or fallback authentication comparisons.
