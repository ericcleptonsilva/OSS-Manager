## 2024-05-18 - [Authentication Bypass] Hardcoded Admin and Password Fallbacks
**Vulnerability:** The codebase contained critical authentication bypasses: `App.tsx` overrode user roles directly based on a matched email (`ericlobobr.01@gmail.com` or `admin@oss.com`), and `services/parseService.ts` featured a legacy login fallback that granted admin privileges if an email/password combination matched hardcoded or unencrypted values (`admin@oss.com` and `admin`).
**Learning:** Legacy authentication methods and conveniences meant for development can introduce severe production vulnerabilities. Relying on client-side hardcoded strings to determine user roles defeats the purpose of the server-side validation.
**Prevention:** Always rely strictly on server-side role validation (e.g., standard Parse `User` and `Role` validation) for privilege assignment. Never use hardcoded emails or default generic passwords as a fallback mechanism for logging in or determining administrative access on the client.
