## 2024-05-30 - [Hardcoded Backdoors and Auth Bypass]
**Vulnerability:** The application contained hardcoded backdoor credentials (`ericlobobr.01@gmail.com` and `admin@oss.com`) in the frontend logic (`App.tsx`, `constants.ts`, and `services/parseService.ts`). Furthermore, the legacy login logic fallback allowed an authentication bypass if the database credentials were not populated due to comparing an empty string against an empty input.
**Learning:** Hardcoded credentials override legitimate authentication flows and pose a critical security risk by providing guaranteed unauthorized access. Additionally, empty string equivalency (`"" === ""`) is a common attack vector when fallback conditionals are improperly written.
**Prevention:** Never hardcode backdoor emails, usernames, or passwords in frontend or backend code. Always explicitly ensure that compared credential variables are truthy (not empty strings or null) before performing direct equality checks during authentication fallbacks.
