## 2024-05-28 - Hardcoded Backdoor Credentials and Privilege Escalation
**Vulnerability:** The application contained hardcoded backdoor email addresses (`ericlobobr.01@gmail.com` and `admin@oss.com`) in `App.tsx` that immediately granted full administrator privileges (`setUserRole('admin')`) to any user logging in with those emails, bypassing normal authentication role checks. Additionally, `parseService.ts` used `admin@oss.com` and `admin` as fallback credentials for Team Admin custom login, allowing authorization bypass. Finally, the default fallback roles for unmatched users and the default React state was insecurely set to `admin`.
**Learning:** Hardcoding administrative credentials or backdoor overrides within frontend code and fallback login procedures exposes the system to trivial privilege escalation and complete system compromise by any user who discovers these values. Fallback logic that defaults to high privileges (like `admin`) violates the principle of least privilege.
**Prevention:** Never hardcode backdoor access or default credentials in source code. Ensure that fallback user roles and default component states strictly adhere to the principle of least privilege by defaulting to the lowest possible access level (e.g., `student`). Validate that custom login flows do not permit access when credentials are empty strings.
