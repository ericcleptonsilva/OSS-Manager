## 2024-05-24 - [CRITICAL] Fixed Hardcoded Backdoor Credentials and Client-Side Secret Leak
**Vulnerability:** A hardcoded admin login backdoor (`admin@oss.com` and `ericlobobr.01@gmail.com`) with a default password (`admin`) was present in `App.tsx` and `services/parseService.ts`. During a previous attempt to fix this, `import.meta.env.VITE_INITIAL_ADMIN_PASSWORD` was used for the fallback authentication. This is a critical vulnerability because Vite injects `VITE_` variables into the client-side JavaScript bundle, exposing the administrative password to all users.
**Learning:** Hardcoded credentials or specific email overrides should never be present in application logic. Furthermore, **never** use `VITE_` prefixed environment variables for sensitive secrets (like passwords or private API keys) in a Vite application, as they will be hardcoded into the public client-side bundle. Authentication fallbacks must not rely on client-side password verification.
**Prevention:** Completely remove client-side backdoor authentication blocks. Enforce that all authentication relies strictly on secure, server-side mechanisms (like `Parse.User.logIn`).
