## 2024-04-03 - [CRITICAL] Fixed Hardcoded Backdoor Credentials
**Vulnerability:** A hardcoded backdoor fallback logic existed in `services/parseService.ts` that allowed an attacker to log in as admin simply by entering `admin@oss.com` and `admin`, bypassing database checks. Furthermore, the `constants.ts` state defaulted to these insecure credentials, and `App.tsx` would forcefully inject them into the initial state if missing or malformed, creating a critical security risk.
**Learning:** Hardcoding administrative fallback credentials or logic within frontend files creates a significant risk, particularly when mixed with real authentication flows. It's unsafe to rely on implicit client-side defaults for roles and permissions.
**Prevention:** Never hardcode secret credentials in source files. Initial data seeds should be left empty or parameterized safely. Authentication fallbacks should only ever query explicit database records or securely bound environment variables, never hardcoded strings inside the authentication flow code. Ensure input validation checks for empty strings to avoid logic bypasses.
