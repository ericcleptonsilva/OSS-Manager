## 2024-05-24 - [Remove Hardcoded Backdoor Credentials and Prevent Empty String Auth Bypass]
**Vulnerability:** The application contained hardcoded admin credentials ('admin@oss.com' and 'ericlobobr.01@gmail.com') which were used as a fallback to grant administrative privileges in `App.tsx` and `services/parseService.ts`. Furthermore, the custom login logic lacked checks for empty strings, potentially allowing authentication bypass if the database had empty credentials.
**Learning:** Hardcoding credentials, even for debugging or fallback purposes, creates a critical security backdoor. When implementing custom fallback authentication mechanisms, it is crucial to explicitly verify that stored and provided credentials are not empty strings. If an empty password is saved in the database, a user providing an empty password string could bypass authentication if equality is the only check.
**Prevention:** Remove all hardcoded credentials from the codebase. Rely on secure, centralized authentication services (like Parse User roles) and ensure that custom credential matching logic validates that both inputs and stored values are truthy and non-empty.
