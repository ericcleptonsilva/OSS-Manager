## 2025-02-18 - Remove Hardcoded Backdoor Admin Login
**Vulnerability:** A hardcoded bypass check for `admin@oss.com` with password `admin` existed directly in the `performCustomLogin` method of `services/parseService.ts`, allowing unauthenticated access to the global admin account regardless of database state. Additionally, `constants.ts` and `App.tsx` were forcing the same hardcoded credentials, preventing the legitimate admin from changing the global admin email.
**Learning:** Hardcoded credentials bypassing normal authentication flow pose a critical risk, especially when forced within UI components (`App.tsx`) and hardcoded directly into auth handlers.
**Prevention:** Never include hardcoded credential checks in authentication services (`performCustomLogin`). Ensure default values like `adminEmail` in initial data are empty or prompt for setup, rather than hardcoding sensitive bypass strings. Ensure frontend inputs for credentials are not set to `readOnly` and do not force hardcoded values on save.
