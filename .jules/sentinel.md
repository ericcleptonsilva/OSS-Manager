## 2026-05-18 - Remove Hardcoded Admin Backdoor
**Vulnerability:** A hardcoded admin credential check and default fallbacks (`admin@oss.com` and `ericlobobr.01@gmail.com` with password `admin`) were embedded in frontend authentication (`App.tsx`) and the custom Parse database fallback query (`services/parseService.ts`), effectively creating a backdoor that could easily bypass normal authentication.
**Learning:** Hardcoded overrides intended for ease of testing or legacy compatibility often slip into production environments. Leaving default fallback passwords (or allowing empty strings) compromises the integrity of custom credential checking flows, allowing unauthenticated privilege escalation to `admin`.
**Prevention:** Rely strictly on database-defined roles (`explicitRole === 'admin'`) and secure authentication providers rather than hardcoded client-side exceptions. When designing custom fallback authentication flows, explicitly enforce that required credential inputs and stored database credentials are non-empty strings before comparison.
