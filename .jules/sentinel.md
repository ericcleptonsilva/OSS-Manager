## 2024-05-24 - Stop Leaking Passwords through Parse.Query
**Vulnerability:** The application was vulnerable to Insecure Direct Object Reference (IDOR) and Data Exposure because sensitive authentication fields (`adminPassword` on `Team` and `Academy` collections) were returned to the client during data fetching functions like `fetchFullData` and `fetchPublicData`.
**Learning:** Standard Parse.Query `.find()` and `.first()` methods retrieve all fields on an object by default, unless specific columns are requested or excluded. Storing authentication fallbacks directly on these data objects meant they were sent in clear text (or at least exposed) to all users interacting with the API unless explicitly excluded.
**Prevention:** Always use `.exclude('sensitiveField')` on Parse queries when querying collections that mix public application data with sensitive operational secrets, especially if the legacy architecture cannot immediately separate those concerns into different collections.
