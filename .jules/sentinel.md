## 2024-05-19 - [CRITICAL] Privilege Escalation via Hardcoded Credentials
**Vulnerability:** The application contained hardcoded backdoor credentials (`admin@oss.com` and `ericlobobr.01@gmail.com`) that unconditionally granted administrator privileges on the frontend, bypassing normal authentication flows. Furthermore, if a standard user did not explicitly have a role defined, they defaulted to `admin` instead of the least-privileged `student`. Lastly, the login fallback assigned `admin` to any user not found in the student or professor list.
**Learning:** Hardcoded credentials and fail-open default roles create multiple critical privilege escalation vectors. A default-allow approach (giving 'admin' role if unknown) is insecure.
**Prevention:** Never hardcode backdoor emails or passwords in production logic. Implement strict "fail-closed" defaults, where users without an explicit role or unknown users default to the lowest privilege level (e.g., `student`). Ensure empty credentials in the database cannot be matched by empty inputs.
