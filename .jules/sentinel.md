## 2025-01-28 - Insecure Direct Object Reference / Information Exposure (Credentials)
**Vulnerability:** The queries in `fetchFullData` and `fetchPublicData` inside `services/parseService.ts` for the `Team` and `Academy` collections were fetching full records from the database without excluding the `adminPassword` field. Because standard Parse queries retrieve all fields unless explicitly excluded or limited with select, this resulted in admin credentials being passed to the client-side JavaScript.
**Learning:** This exposes the platform to credential theft since any user loading the app could view the API response payloads and obtain the legacy fallback passwords stored in `adminPassword` fields for both the Team (global admin) and Academies (instructors).
**Prevention:** Sensitive fields that are verified to exist (such as `adminPassword` on `Team` and `Academy` objects in this schema) MUST be explicitly excluded using `.exclude('fieldName')` from Parse Server queries everywhere they are called except for specific server-side authentication functions (e.g. `performCustomLogin`) where they are absolutely necessary to be evaluated.
