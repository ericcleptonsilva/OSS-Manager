## 2024-05-18 - Privilege Escalation via Hardcoded Admin Backdoor
**Vulnerability:** The application contained hardcoded backdoor email addresses (`admin@oss.com`, `ericlobobr.01@gmail.com`) in the frontend logic (`App.tsx`) that automatically escalated the user's role to `admin` without verifying any password. Furthermore, if a user logged in without a clear role, it defaulted to `admin` instead of least-privilege `student`. Empty string passwords could also potentially bypass authentication in `services/parseService.ts`.
**Learning:** Hardcoded backdoors and explicit UI-side role assignments overriding database permissions are severe security risks, especially when they act as an "override" for any given user. Fallback assignments must always default to the least privileged role.
**Prevention:** Remove all hardcoded backdoor credentials. Ensure fallback role assignments default to `student`. Ensure custom authentication explicitly checks that stored and provided credentials are not empty strings.
