## 2024-05-24 - Remove Hardcoded Admin Backdoor Credentials\n**Vulnerability:** Hardcoded email addresses (`admin@oss.com`, `ericlobobr.01@gmail.com`) and passwords (`admin`) were present in `App.tsx` and `services/parseService.ts`. This allowed anyone providing these credentials to bypass normal authentication and gain full administrator privileges.\n**Learning:** Default configuration values or placeholder test credentials left in production code create critical backdoors. Furthermore, relying on client-side hardcoded strings for authorization checks (`isAdminEmail` boolean flag) is inherently insecure.\n**Prevention:** Never hardcode credentials or specific user identifiers in code for authorization checks. Ensure fallback/legacy authentication logic strictly checks that stored credentials are not empty strings before attempting comparison, preventing authentication bypasses if the database is uninitialized.
