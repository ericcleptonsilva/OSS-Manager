## 2025-02-23 - Hardcoded Admin Credentials and Authorization Bypass
**Vulnerability:** The application contained hardcoded admin credentials (`admin@oss.com` and `ericlobobr.01@gmail.com`) that granted automatic administrative access bypassing actual role verification. Furthermore, empty fallback credential strings in the database allowed an attacker to bypass authentication checks during custom login if default configurations weren't updated.
**Learning:** Hardcoded overrides intended for testing or default states frequently leak into production if not strictly managed. The custom fallback authentication mechanism (`performCustomLogin`) lacked explicit checks to prevent empty string matches against uninitialized database fields, creating an implicit backdoor.
**Prevention:** Always mandate strict existence checks (`!== undefined`, `!== null`, `!== ''`) for retrieved credentials before performing validation. Never include fallback override logic for specific emails in the client-side authorization routing.
