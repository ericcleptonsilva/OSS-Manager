## 2025-04-29 - Hardcoded Credential Bypass Removed
**Vulnerability:** Hardcoded admin emails ('ericlobobr.01@gmail.com', 'admin@oss.com') were automatically mapped to full admin privileges in App.tsx. Furthermore, parseService.ts fallback authentication permitted login without checking if stored admin credentials were empty strings, allowing authentication bypass when the database had no password saved. Default privileges were escalated to 'admin' in catch-blocks.
**Learning:** Hardcoded credentials should never exist in production source code, especially when used for overriding normal authorization flows. Empty fields returned from database lookups can be incorrectly evaluated as matching user inputs if strict truthiness isn't checked in legacy fallback logic.
**Prevention:** Always enforce role defaults to least privilege ('student'). Strip out explicit overrides of application variables. Ensure strict truthiness checks (if (storedPass && ...)) when comparing secrets to avoid empty-string evaluation exploits.
