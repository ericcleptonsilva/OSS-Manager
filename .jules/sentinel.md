## 2024-04-14 - Fix Hardcoded Credentials and Privilege Escalation
**Vulnerability:** Hardcoded admin emails (`admin@oss.com`, `ericlobobr.01@gmail.com`) and default passwords (`admin`) were used to automatically grant `admin` privileges in the frontend state (`App.tsx`) and the custom fallback login routine (`services/parseService.ts`). In addition, missing role attributes incorrectly defaulted to `admin` instead of the least-privileged `student` role. Empty string authentication bypasses were possible when the credentials fell back to empty strings without validation.
**Learning:** Hardcoding credentials and injecting admin bypasses directly in the application bypasses proper server-side authentication and role management mechanisms (like Back4App's Parse roles). Furthermore, assuming high privileges (admin) upon a failure to detect a role explicitly breaks the principle of least privilege.
**Prevention:** Always authenticate explicitly with a robust identity provider without hardcoded bypasses. Initialize default roles to the lowest permission level (e.g., `student`). Reject empty string comparisons in authentication fallback pathways, ensuring variables are explicitly validated before granting access.
