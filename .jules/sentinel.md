## 2024-04-04 - Hardcoded Backdoor Credentials and Empty-String Login Bypass
**Vulnerability:** A hardcoded legacy admin fallback was present in `services/parseService.ts` (`admin@oss.com` / `admin`), allowing unauthorized access. When replacing this with environment variable fallbacks, it introduced an empty-string login bypass risk if the variables are undefined and not strictly checked.
**Learning:** Hardcoded credentials should never be committed to the repository, especially not as a global administrative backdoor fallback. In addition, when extracting credentials to environment variables, missing variables might resolve to empty strings. If the authentication logic strictly compares against an empty string without explicitly checking for truthiness or emptiness, a user could login by entering an empty email and password.
**Prevention:** Never use hardcoded legacy fallbacks for authentication. Ensure all authentication comparisons explicitly check and reject empty strings (`!== ''`) before executing string matches against stored or fallback credentials.
