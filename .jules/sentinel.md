## 2025-05-18 - [CRITICAL] Fix hardcoded backdoor credentials
**Vulnerability:** A hardcoded credentials backdoor existed in the `performCustomLogin` auth logic allowing unauthenticated access to the `admin` role via 'admin@oss.com' and the password 'admin'. Additionally, `App.tsx` possessed hardcoded logic that overwrote admin credentials to match this backdoor upon editing team settings, and `constants.ts` contained this static credential pair in its `INITIAL_DATA`.
**Learning:** This reveals a known vulnerability pattern within the codebase where developers re-introduce hardcoded credentials to ease testing or debugging, circumventing robust authentication controls.
**Prevention:** Eliminate all instances of hardcoded global fallback credentials (`admin@oss.com`/`admin`) in `services/parseService.ts`, `App.tsx`, and `constants.ts`. Instead, dynamically query properly secured and hashed authentication records via Parse, and enforce rigorous code reviews to deter the re-introduction of debugging backdoors.
