## 2024-05-24 - Remove Hardcoded Backdoor Credentials
**Vulnerability:** A backdoor authentication bypass logic existed in `services/parseService.ts` (`performCustomLogin`) allowing login with 'admin@oss.com' and 'admin' regardless of what was stored in the database. Furthermore, `App.tsx`'s `handleEditTeam` would force setting these credentials if they were absent, and `constants.ts` initialized the DB with them.
**Learning:** Hardcoded credentials acting as fallback or testing mechanisms often slip into production code and expose systems to trivial takeover. This pattern of falling back to default, static admin credentials is a critical security risk.
**Prevention:** Remove all hardcoded credentials from code, including authentication fallbacks and initial seed data logic. Rely on proper database seeding tools and securely managed environment variables or secure vault setups for initializing an admin account if necessary. Ensure no "test" credentials override standard authentication checks.
