## 2025-03-03 - [Auth Bypass & Hardcoded Backdoors]
**Vulnerability:** The application contained hardcoded admin emails (`admin@oss.com`, `ericlobobr.01@gmail.com`) that bypassed authentication logic and forced admin access. Additionally, empty string values for stored passwords in custom login logic could allow an attacker to bypass authentication if they provided an empty string password.
**Learning:** Hardcoding 'known' emails in frontend client code creates permanent backdoors. Furthermore, fallback mechanisms (like checking legacy database fields for passwords) must enforce that the stored values are actually set (not empty or falsy strings) before allowing a match, otherwise users with missing passwords can be logged into.
**Prevention:** Never use hardcoded identifiers to assign elevated roles. Rely strictly on roles fetched from the authentication provider. When comparing credentials against fallback mechanisms, always explicitly verify that the fetched credentials are truthy/non-empty strings before performing equality checks.
