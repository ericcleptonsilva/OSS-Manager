## 2024-05-24 - Hardcoded Admin Credentials and Default Privilege Escalation
**Vulnerability:** The application contained hardcoded backdoor administrator emails (`ericlobobr.01@gmail.com` and `admin@oss.com`) and passwords (`admin`) that would silently grant admin access on the client side (`App.tsx`) and via an authentication override in the mock auth fallback (`services/parseService.ts`). Additionally, unauthenticated users or those without defined roles defaulted to `admin` instead of least-privileged `student`.
**Learning:** Hardcoded credentials combined with default-allow/default-admin role resolution creates a massive authentication bypass and privilege escalation risk. Even when intended as a fallback or placeholder, sensitive secrets and global admin overrides should never be baked into source code.
**Prevention:** Default all UI states, undefined role states, and catch-blocks to the lowest privilege (`student`). Explicitly ensure that authentication queries verify actual credentials rather than falling back to string literals. Remove any hardcoded admin bypass routes in the front end.
