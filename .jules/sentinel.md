## 2025-05-08 - Authentication Bypass and Hardcoded Backdoors
**Vulnerability:** The custom authentication logic (`services/parseService.ts`) permitted logging in via fallback Parse objects (`Team` or `Academy`) without verifying if the retrieved fallback credentials were empty. If a `Team` or `Academy` was created without `adminEmail` or `adminPassword`, these fields defaulted to empty strings, allowing an attacker to bypass authentication by providing empty login fields. Additionally, the frontend (`App.tsx`) contained hardcoded email backdoors (`ericlobobr.01@gmail.com` and `admin@oss.com`) that automatically granted 'admin' roles to users, bypassing the database role assignments entirely. There was also UI logic re-inserting default 'admin' passwords.
**Learning:** Hardcoded overrides break the role-based access control (RBAC) model. Client-side authentication logic must be robust enough to handle malformed or missing fallback credentials from a remote datastore to prevent logic errors like empty-string bypasses.
**Prevention:** Always ensure fallback authentication checks explicitly enforce minimum constraints (e.g., non-empty strings) on credentials retrieved from the database before performing comparisons. Never implement 'super-user' overrides directly into frontend state management logic; use backend roles strictly.
